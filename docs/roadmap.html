プロジェクト概要

食材をキャラクター化し、AIによる賞味期限管理とレシピ提案で食品ロスを削減する、楽しく使える家族向け冷蔵庫管理アプリの開発プロジェクトです。

3人機能別フルスタック分担

開発者A：バーコード・AI認識機能

機能範囲: バーコード読込→画像認識→商品データ取得→賞味期限予測

フルスタック担当:

  • 📱 UI: スキャナー画面、結果表示、エラー画面
  • 🔧 ロジック: バーコード処理、Gemini AI統合
  • 🗃️ データ: 商品DB、OCR結果、AI応答処理
  • ☁️ API: Open Food Facts、Gemini、画像処理

担当ディレクトリ:
lib/features/scanner/
lib/core/services/gemini_service.dart
lib/core/services/ocr_service.dart

開発者B：冷蔵庫管理機能

機能範囲: 冷蔵庫表示→商品管理→賞味期限アラート→家族共有

フルスタック担当:

  • 📱 UI: ホーム画面、商品詳細、家族共有画面
  • 🔧 ロジック: 商品管理、期限計算、通知処理
  • 🗃️ データ: 商品Firestore、家族データ同期
  • ☁️ API: 家族招待、リアルタイム同期

担当ディレクトリ:
lib/features/home/
lib/features/products/
lib/features/household/

開発者C：認証・設定・履歴・キャラクター機能

機能範囲: ユーザー認証→設定管理→履歴分析→キャラクター画像生成

フルスタック担当:

  • 📱 UI: ログイン、設定、履歴、統計、キャラクター表示画面
  • 🔧 ロジック: 認証処理、設定管理、データ分析、キャラクター感情制御
  • 🗃️ データ: ユーザーDB、設定データ、履歴データ、キャラクター画像キャッシュ
  • ☁️ API: Firebase Auth、Analytics、通知、Vertex AI Imagen
  • 🎨 画像生成: 賞味期限連動キャラクター感情表現（新鮮→困り顔→ゾンビ化）

担当ディレクトリ:
lib/features/auth/
lib/features/settings/
lib/features/history/
lib/core/services/imagen_service.dart

Git戦略・開発フロー

ブランチ戦略 (Git Flow)

main                    # 本番リリース (App Store/Play Store)
├── develop             # 開発統合・ステージング
    ├── feature/ui-*    # フロントエンド機能開発
    ├── feature/ai-*    # AIバックエンド機能開発
    └── feature/data-*  # データ・インフラ機能開発

開発プロセス

  1. Feature開発: develop から feature/xxx ブランチ作成
  2. コードレビュー: PR作成・最低1名レビュー必須
  3. 統合テスト: develop マージ後自動テスト実行
  4. リリース: develop → main で本番デプロイ
